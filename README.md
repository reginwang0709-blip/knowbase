# 更新概述
本次更新将知识库从静态模拟演示版本，升级为支持小宇宙播客链接的真实解析工作流。

# 已实现功能
- 新增基于 Supabase 的持久化存储，支持解析任务、内容、知识库分类、话题及内容-话题关联关系。
- 首页提交流程已对接 API 接口与数据库记录。
- 新增重复链接处理机制：
  - 解析中的任务返回 `TASK_IN_PROGRESS`
  - 已完成解析的内容返回 `CONTENT_ALREADY_EXISTS`
  - 失败任务支持重新解析
- 新增单条内容删除流程。
- 新增小宇宙单集元数据提取功能：
  - 标题
  - 作者/主播
  - 发布时间
  - 简介/节目笔记
  - 封面图链接
  - 音频链接
- 接入 Fun-ASR 异步语音转文字流程：
  - 提交 ASR 任务
  - 将 ASR 任务状态存入 `parse_tasks.processing_payload`
  - 通过 `GET /api/parse-tasks/[id]` 轮询任务状态
  - 将真实语音转文字段落写入 `contents.content_payload`
- 重构首页任务卡片，展示真实任务状态与进度，替代前端模拟进度。
- 避免解析失败时自动降级加载演示内容。
- 收紧 `/content/[id]` 降级逻辑：
  - 保留 `/content/demo-001` 作为演示地址
  - 真实内容 ID 出错时不再自动降级为演示内容
- 优化知识包展示效果：
  - 界面中摘要截断为 200 字符
  - 带时间戳的节目笔记可生成语义化章节
  - 语音转文字段落经清洗与合并，提升可读性

# 当前局限
- 基于大模型的知识包生成功能尚未接入。
- 当前真实解析仅覆盖元数据、音频链接提取、ASR 转文字及基于节目笔记的章节划分。
- 关键词、术语表及深度语义摘要需在下一阶段由大模型生成。
- 小宇宙解析依赖对节目页面与音频链接的正常访问。
- ASR 调用需谨慎控制，会消耗外部接口配额。

# 后续计划
3G 阶段将以现有真实语音转文字段落为输入，通过大模型生成真实内容：
- 内容摘要
- 章节划分
- 关键词
- 术语表

完成后，3H 阶段将接入基于大模型的知识库分类与话题分配功能。
