# KnowBase Final Product Spec

## 1. 产品定位

KnowBase 是一个单链接输入的个人知识提取与知识管理工具。

用户只需要粘贴一个内容链接，系统自动完成内容解析、逐字稿生成、语义章节切分、关键词解释、原文证据锚定，并将结果沉淀进个人知识库。

KnowBase 的核心价值不是“把内容转成文字”，而是把碎片内容转化为可回看、可搜索、可归档、可关联的结构化知识。

## 2. 核心产品主张

一个链接进去，生成一条可沉淀的知识包。

知识包不是临时结果，而是会进入用户的知识库结构中，成为后续搜索、回看、归档和关联的基础。

## 3. 用户目标

用户希望解决：

1. 听过或看过很多内容，但很难沉淀。
2. 节目 shownotes 只能服务单期内容，不能形成个人知识库。
3. 内容消费后无法搜索、归档和跨内容关联。
4. 想快速理解专有名词、关键概念和原文语境。
5. 想把长期输入变成可管理的知识资产。

## 4. 产品核心链路

用户粘贴链接
→ 系统识别来源
→ 系统提取内容
→ 系统生成逐字稿或正文
→ 系统按语义切分章节
→ 系统生成知识包
→ 系统提取关键词、解释和原文证据
→ 内容进入知识库
→ 用户在知识库中搜索、回看和进入详情

## 5. 首发支持范围

产品首发优先支持：

1. 小宇宙
2. YouTube
3. 网页文章

其他平台作为后续扩展方向。

## 6. 核心页面

### 首页 /

首页负责提交链接和展示任务处理状态。

核心能力：

- 单链接输入
- 任务状态
- 解析进度
- 最近任务入口
- 进入知识库入口

### 知识库页 /library

知识库页负责展示用户的知识结构。

核心能力：

- 最近高频主题
- 动态生成的一级分类卡片
- 二级主题结构
- 最近解析内容
- 搜索
- 后续可扩展图谱视图、表格视图、思维导图视图

### 知识包页 /content/[id]

知识包页负责展示单条内容的结构化知识结果。

核心能力：

- 内容元信息
- 内容摘要
- 目录 / 语义章节导航
- 语义章节
- 章节摘要
- Top 5 关键词
- 关键词解释
- 原文证据
- 逐字稿 / 正文
- 后续可扩展批注、高亮、导出

## 7. 关键概念

### 知识包 Knowledge Pack

知识包是一条内容被系统处理后的结构化结果。

一个知识包至少包含：

- 标题
- 来源平台
- 来源链接
- 作者 / 发布者
- 发布时间
- 解析时间
- 内容摘要
- 语义章节
- 章节摘要
- Top 5 关键词
- 关键词解释
- 原文证据
- 逐字稿或正文

知识包页必须包含目录。目录由系统自动切分的语义章节生成，用于帮助用户快速理解长内容结构。用户点击目录项后，可以跳转到对应章节、摘要或逐字稿段落。

目录不是手动写死的导航，而是知识包结构的一部分。

### 知识库 Library

知识库不是普通内容列表，而是用户长期积累内容后的组织结构。

知识库应体现：

- 内容被归档到知识领域
- 主题被聚合
- 高频概念被展示
- 用户可以搜索和回看
- 用户可以从分类进入具体知识包

### 动态归档 Dynamic Archive

KnowBase 的知识库不使用固定预设分类。

一级分类和二级主题应随着用户内容积累动态生成和调整。

系统会根据内容摘要、关键词、语义章节、原文证据和语义相似度，判断新内容应进入已有主题、建议新建主题，或在必要时建议新建一级分类。

分类、主题、关键词三者不能混用：

- 一级分类：由多个主题聚合形成的稳定知识领域
- 二级主题：由多条内容和关键词聚合形成的具体知识方向
- 关键词：从单条内容中提取的核心概念，用于解释、搜索和关联

系统可以提出合并、拆分、重命名分类的建议，但必须由用户确认。用户手动调整结果优先于 AI 自动建议。

## 8. 产品原则

1. 最低用户交互成本：用户只需要粘贴一个链接。
2. 结构化结果优先：KnowBase 不只是生成文字，而是生成结构化知识结果。
3. 知识管理优先于一次性生成：系统生成的内容必须进入知识库。
4. 原文可信度优先：逐字稿层不应被生成式改写。
5. 技术阶段可降级，产品动线不可降级。
6. 动态归档优先：知识库结构应由用户内容生长出来，而不是从固定分类模板开始。
