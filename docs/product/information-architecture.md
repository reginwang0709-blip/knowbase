# KnowBase Information Architecture

## 1. 总体页面动线

KnowBase 的核心动线是：

首页提交链接
→ 任务处理
→ 生成知识包
→ 存入知识库
→ 用户在知识库中搜索、浏览、回看
→ 点击具体内容进入知识包页

## 2. 路由结构

### /

首页，也是任务入口。

功能：

- 粘贴链接
- 提交解析任务
- 展示任务进度
- 显示完成后的知识包入口
- 提供进入知识库入口

首页不是完整知识库，也不是结果页。

### /library

知识库页。

功能：

- 展示用户知识结构
- 展示最近高频主题
- 展示动态生成的一级分类卡片
- 展示分类下的二级主题和内容
- 支持搜索
- 提供进入知识包页的入口

知识库页不是普通解析历史列表。

### /content/[id]

知识包页。

功能：

- 展示某条内容的结构化知识结果
- 展示摘要、章节、关键词、证据、逐字稿
- 用户可以从这里理解内容，并回到知识库

知识包页不是普通逐字稿页。

## 3. 首页结构

首页应包含：

1. 产品标题和说明
2. 链接输入区
3. 任务状态卡片
4. 最近任务或完成任务入口
5. 知识库入口

首页核心感受：

用户粘贴一个链接后，能清楚知道系统正在处理什么、处理到哪一步、完成后去哪里查看。

## 4. 知识库页结构

知识库页是 KnowBase 的核心组织层。

默认结构：

1. 页面标题和说明
2. 搜索框
3. 最近高频主题
4. 一级分类卡片区
5. 分类详情展开区
6. 最近解析内容辅助区

知识库页的主体是一级分类卡片，而不是内容列表。

## 5. 知识库层级

知识库页应体现三层结构：

一级分类
  └── 二级主题
        └── 具体内容 / 知识包

一级分类和二级主题不是系统固定模板，而是系统根据用户已解析内容动态聚合生成的知识结构。

页面中展示的分类卡片应表达以下含义：

- 该分类由若干内容自动聚合形成
- 该分类下包含若干二级主题
- 二级主题由多条内容和关键词聚合形成
- 关键词用于搜索、解释和关联，不直接等于分类

## 6. 知识库页默认内容

默认进入 /library 时，应优先展示：

1. 最近高频主题
2. 一级分类卡片
3. 最近解析内容

不应默认把所有内容平铺为普通列表。

一级分类卡片应展示 generatedReason、sourceContentCount 或类似信息，用于说明该分类是如何由内容聚合形成的。

## 7. 知识包页结构

知识包页应包含：

1. 顶部返回与导航
2. 内容元信息
3. 内容摘要
4. 目录 / 语义章节导航
5. 语义章节
6. Top 5 关键词
7. 逐字稿 / 正文
8. 原文证据定位

页面应让用户先看到结构化结果，再进入长文本阅读。

## 知识包页目录

知识包页必须包含目录。

目录来源于系统自动切分的语义章节，而不是页面手写固定导航。

桌面端建议将目录放在左侧栏或内容区顶部，便于用户在长内容中快速跳转。

移动端可以将目录折叠为顶部导航、下拉目录或抽屉。

目录项至少包含：

- 章节标题
- 章节摘要的简短提示
- 对应的起始时间戳或 transcript block 锚点

目录交互：

点击目录项
→ 页面滚动到对应章节或逐字稿段落
→ 当前章节在目录中高亮
→ 对应 transcript block 可以短暂高亮

知识包页不能只有摘要、关键词和逐字稿。目录是长内容阅读体验的核心。

## 8. 证据锚点交互

关键词、章节和摘要中的关键结论应能定位到对应逐字稿段落。

交互方式：

点击“查看原文证据”
→ 页面滚动到 transcript block
→ 对应 block 短暂高亮

## 9. 搜索范围

知识库搜索应覆盖：

- 一级分类名称
- 二级主题名称
- 内容标题
- 关键词
- 后续阶段扩展到全文搜索
