# Phase 1A — Library Page Redesign

## 背景

当前知识库页不能做成普通内容卡片列表。

KnowBase 的知识库页需要体现：

- 内容被归档进知识结构
- 主题被聚合
- 分类可以展开
- 用户可以搜索和回看
- 用户能从知识结构进入具体知识包

知识库页的核心不是“展示所有解析内容”，而是“展示用户的知识结构”。

## 页面目标

重设计 /library 页面，使其成为知识库结构页。

用户进入 /library 后，应看到：

1. 最近高频主题
2. 一级分类卡片
3. 点击一级分类后看到二级主题和内容
4. 最近解析内容作为辅助入口
5. 搜索可以搜索分类、主题、内容标题和关键词

## 页面整体结构

页面从上到下：

1. 页面标题区
2. 搜索区
3. 最近高频主题区
4. 一级分类卡片区
5. 分类详情展开区
6. 最近解析内容区

## 页面标题区

展示：

- 标题：知识库
- 说明：系统会将解析后的内容自动归入知识结构，方便回看和整理

不要把页面标题写成“解析历史”或“内容列表”。

## 搜索区

搜索框 placeholder：

搜索标题、关键词或分类

搜索范围：

- 一级分类名称
- 二级主题名称
- 内容标题
- 关键词

搜索结果可以按类型分组展示：

- 匹配的分类
- 匹配的主题
- 匹配的内容

## 最近高频主题区

展示 5 个主题 chip：

- AI Agent
- RAG
- 知识库
- 产品经理
- 工作流

每个 chip 展示：

- 主题名称
- 相关内容数量

点击 chip 后：

- chip 高亮
- 下方内容根据主题过滤
- 显示“清除筛选”按钮

## 一级分类卡片区

这是知识库页的主体。

默认展示一级分类卡片，而不是直接展示所有内容。

每张一级分类卡片包含：

- 一级分类名称
- 分类简介
- 内容数量
- 二级主题数量
- 高频关键词
- 最近更新时间
- “查看分类”按钮

示例一级分类：

AI 产品：围绕 AI Agent、RAG、智能工作流和 AI 产品设计的内容。
产品方法论：围绕需求分析、MVP、用户反馈和产品判断的内容。
技术理解：围绕 API、数据库、异步任务和前后端架构的内容。

## 分类详情展开区

用户点击一级分类卡片后，在当前页面展开分类详情。

不要直接跳转到内容页。

分类详情包含：

- 当前一级分类名称
- 分类简介
- 二级主题列表
- 每个二级主题下的内容列表

二级主题卡片包含：

- 二级主题名称
- 内容数量
- 高频关键词
- 内容列表

内容列表项包含：

- 内容标题
- 来源平台
- 作者
- Top 3 关键词
- 摘要一行
- 解析时间
- 点击后跳转 /content/demo-001

示例结构：

AI 产品

Agent 产品
- 为什么 Agent 不是 Chatbot
- 企业 Agent 的落地路径

RAG 与知识库
- RAG 为什么适合企业知识管理
- 向量数据库与检索增强

AI 工作流
- 从 Workflow 到 Agentic Workflow

## 最近解析内容区

最近解析内容是辅助入口，不是页面主体。

展示最近 3 到 5 条内容。

每条内容包含：

- 标题
- 来源平台
- 所属分类路径
- Top 3 关键词
- 解析时间
- 点击进入知识包页

## 交互规则

### 点击一级分类

- 当前分类卡片高亮
- 页面内展开分类详情
- 页面滚动到分类详情区域

### 点击内容

跳转到：

/content/demo-001

### 点击高频主题

- 过滤相关分类和内容
- 高亮当前主题
- 显示清除筛选

### 搜索

- 输入关键词后过滤分类、主题和内容
- 搜索为空时恢复默认知识库结构

## 数据要求

需要在 src/lib/mock-data.ts 中提供：

- recentTopics
- libraryCategories
- recentContents

数据结构参考：

docs/contracts/data-contracts.md

## 当前阶段不要实现

不要实现：

- 真实归档算法
- 真实 API
- Supabase
- 图谱视图
- 思维导图视图
- 表格视图
- 拖拽排序
- 编辑分类
- 批注
- 登录

## 验收标准

- /library 默认展示一级分类卡片
- 页面顶部有最近高频主题
- 搜索可以过滤分类、主题、内容
- 点击一级分类后能看到二级主题和内容列表
- 最近解析内容只是辅助区域
- 内容点击后能进入 /content/demo-001
- 所有内容来自 mock data
- 没有新增真实 API 或数据库依赖
