# Phase 1 — Mock Frontend

## 阶段目标

用 mock data 做出 KnowBase 的三页可点击产品壳。

当前阶段不接真实 API，不接数据库，不接 ASR，不接 LLM。

但页面动线必须对齐终版产品方向：

链接输入
→ 任务处理
→ 知识包
→ 知识库沉淀
→ 回看与搜索

## 页面范围

当前阶段实现三个页面：

1. 首页 /
2. 知识库页 /library
3. 知识包页 /content/[id]

## 首页 /

首页目标：

让用户理解 KnowBase 的入口是“粘贴一个链接”，并看到任务处理状态。

首页需要包含：

- 产品名称
- 产品说明
- 小宇宙链接输入框
- “开始解析”按钮
- mock 任务卡片
- 任务状态进度条
- 完成后进入知识包页的按钮
- 进入知识库的入口

任务状态顺序：

1. 已提交
2. 正在识别链接来源
3. 正在提取内容信息
4. 正在生成逐字稿
5. 正在生成知识包
6. 已完成

## 知识库页 /library

知识库页目标：

让用户感觉内容已经被归档、聚合和沉淀，而不是只看到解析历史。

默认结构应包括：

1. 页面标题与说明
2. 搜索框
3. 最近高频主题
4. 一级分类卡片
5. 分类详情展开区
6. 最近解析内容

知识库页不要做成普通内容列表。

具体设计以 docs/phases/phase-1a-library-redesign.md 为准。

## 知识包页 /content/[id]

知识包页目标：

展示一条内容被结构化后的结果。

需要包含：

- 内容标题
- 来源平台
- 来源链接
- 作者 / 发布者
- 发布时间
- 解析时间
- 内容摘要
- 语义章节
- 灰色斜体章节摘要
- Top 5 关键词
- 关键词解释
- 当前内容中的语境示例
- 原文证据入口
- 逐字稿 block

- ASR
- LLM
- 真实 API
