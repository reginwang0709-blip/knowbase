# Historical Task — Library Dynamic Archive

## 任务名称

将 `/library` 从普通内容列表重构为动态归档知识库结构页。

## 任务背景

KnowBase 的知识库页不应只是解析历史或内容列表，而应该表达用户内容被持续归档、聚合和沉淀后的知识结构。

分类不是系统固定模板，而是根据用户已解析内容动态生成的知识领域。

## 已实现重点

- `/library` 默认主体从内容平铺列表调整为知识领域结构。
- 增加“最近关注”主题入口，用于按高频主题筛选内容。
- 增加知识领域卡片，展示领域名称、简介、内容数量、结构置信度、高频关键词等信息。
- 增加领域详情区，点击领域后在当前页面展开二级主题和内容列表。
- 增加最近新增内容区域，作为辅助回看入口，而不是页面主体。
- 搜索覆盖领域、主题、内容标题、关键词和作者等信息。
- 页面文案表达“内容会被整理进知识领域，并随新增内容和用户调整持续变化”。
- 避免把知识领域表现成固定分类模板。

## 数据原则

- 使用同一份 `libraryCategories`、`recentTopics`、`recentContents` mock 数据。
- 分类数据作为“动态生成结构的示例”，不是系统内置固定分类。
- 内容点击统一进入 `/content/demo-001`，不扩展真实多内容详情页。

## 范围控制

未实现真实动态归档算法、真实 API、数据库、Supabase、图谱、批注、导出或登录。
